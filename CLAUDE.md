## Project Overview

**xtea_cython** is a high-performance XTEA encryption library implemented in Cython. It provides multiple encryption modes (ECB, CBC, CFB, OFB, CTR) and is designed as a drop-in replacement for the PyPI `xtea` library with significant performance improvements.

## Key Architecture

```
xtea_cython/
├── core.pyx          # Cython XTEA core (8-byte blocks, 16-byte key, big-endian)
├── modes.py          # Encryption modes (ECB, CBC, CFB, OFB, CTR)
├── batch.pyx         # Batch processing wrapper (interleaved execution)
├── xtea_batch.c      # C implementation for interleaved batch processing
├── xtea_batch.h      # Header for batch functions
├── __init__.py       # Public API exports
├── setup.py          # Build configuration with platform-specific optimizations
├── pyproject.toml    # Project metadata and cibuildwheel config
└── tests/            # Test suite (53 tests)
```

## Important Technical Details

### XTEA Algorithm
- Block size: 8 bytes (64 bits)
- Key size: 16 bytes (128 bits)
- Default rounds: 32 (64 Feistel rounds)
- Byte order: Big-endian (for PyPI xtea compatibility)

### Security Features
- Constant-time PKCS#7 padding validation (timing attack protection)
- SecurityWarning for ECB mode usage
- SecurityWarning for CTR nonce reuse
- Counter wrap-around detection in CTR mode
- Minimum rounds validation (8 rounds minimum recommended)

### Performance Optimizations
- Cython with `nogil`, `boundscheck=False`, `wraparound=False`
- Platform-specific compiler flags (-O3, -march=native, -funroll-loops)
- Batch processing with interleaved execution (6x faster than single-block)

## API Reference

```python
from xtea_cython import (
    # Helpers
    generate_key,    # generate_key(size=16) -> bytes
    generate_iv,     # generate_iv(size=8) -> bytes
    BLOCK_SIZE,      # = 8

    # Modes (recommended first)
    encrypt_cbc, decrypt_cbc,    # CBC - recommended
    encrypt_ctr, decrypt_ctr,    # CTR - stream mode
    encrypt_cfb, decrypt_cfb,    # CFB - stream mode
    encrypt_ofb, decrypt_ofb,    # OFB - stream mode
    encrypt_ecb, decrypt_ecb,    # ECB - NOT secure for most uses
    encrypt, decrypt,            # Backward compat (CBC)

    # Padding
    pkcs7_pad, pkcs7_unpad,

    # Low-level
    encrypt_block, decrypt_block,  # Single 8-byte blocks

    # Security
    SecurityWarning,
)

# Batch processing (4x faster for bulk operations via interleaved execution)
from xtea_cython.batch import encrypt_blocks_batch, decrypt_blocks_batch
```

## Known Issues

1. **Cython .c files generated at build time**: `core.c` and `batch.c` are auto-generated by Cython during build. Do not commit them.

2. **Platform-specific flags**: The setup.py has different compiler flags for macOS, Linux, and Windows. Be careful when modifying.

3. **PyPI xtea compatibility**: Always maintain big-endian byte order for compatibility with the reference implementation.

## Testing Strategy

- All modes must have roundtrip tests
- Edge cases: empty data, partial blocks, multiple blocks
- Validation tests: invalid key/IV lengths
- Counter wrap-around test for CTR mode
- Compatibility tests with PyPI xtea library (optional, requires `pip install xtea`)

## Release Process

1. Update version in `pyproject.toml` and `__init__.py`
2. Ensure all tests pass
3. Create GitHub release: `gh release create v0.x.x`
4. Wheels are automatically built and published to PyPI
